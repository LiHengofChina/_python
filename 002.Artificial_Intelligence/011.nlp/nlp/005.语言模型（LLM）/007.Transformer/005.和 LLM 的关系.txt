//====================================== Token
Token	//文本切成小单元，是模型输入单位
		/**
			Token = 把文本切成模型能处理的小单元（字、词、子词都行），是最基础的切分。
			分词、词性标注是传统 NLP 的额外步骤；大模型一般只做 Tokenization，不做词性标注。
		*/


//====================================== Embedding
//====================================== Embedding
Embedding 	//每个Token变成向量，送
		/**
			词嵌入（word embedding）。
			以及 005.语言模型 里的 Word2Vec。
		*/


//====================================== 上下文窗口
//====================================== 上下文窗口
一次最多能看多少个 Token（如 4K、128K）


//====================================== 生成
//====================================== 生成

Decoder 每次预测下一个 Token，循环输出
		/**
			Decoder 负责生成输出：
				根据已有内容，每次预测下一个 Token，
				再把这个 Token 接上去，继续预测，直到结束。
				例如输入「今天天气」→ 预测「很好」→ 再预测「」→ 结束。
				所以 Decoder = 往外“写”内容的那一半；
				Encoder 是 理解输入 的那一半（GPT 类模型通常没有 Encoder，只有 Decoder）（ GPT 和 ChatGPT 都是这样）。
		*/


//========================================================== 总结流程（整个 LLM 生成流程；Transformer 只是中间核心一步：）
//========================================================== 总结流程（整个 LLM 生成流程；Transformer 只是中间核心一步：）


流程： 文本 → 切 Token → Embedding → Transformer → 预测下一个词 → 循环

（1）切 Token				Tokenizer（Tokenizer 工具）
（1）Embedding				把 Token 变向量
（1）Transformer				核心计算（理解+预测）
（1）预测下一个词 → 循环		自回归生成（Decoder 行为）

//Transformer 是心脏，前后还有 Token、Embedding 和循环生成。

//==========================================================
