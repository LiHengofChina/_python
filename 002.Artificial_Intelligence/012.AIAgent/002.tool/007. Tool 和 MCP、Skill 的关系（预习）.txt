Tool     = 单个函数（ssh_exec、rag_search）
MCP      = 用统一协议暴露一批 Tool/资源（跨应用、跨 IDE）
Skill    = 更高层：指令 + 流程 + 可能多个 Tool 的组合（SOP）

你平台里：先实现 Tool，后面用 MCP 标准化暴露，
		再用 Skill 封装「新增规则」「故障排查」等流程。

//================================================================== Tool
//================================================================== Tool

是什么 单个能力（SSH、查库、RAG）	
粒度： 原子操作	
例子： ssh_exec("df -h")

//================================================================== Skill
//================================================================== Skill

是什么: 一套可复用流程/能力包
粒度： 多步骤 + 指令 + 可能多个 Tool	
例子： 「新增识别规则」整套检查清单
