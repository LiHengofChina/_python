//=========================================================== 答1

Demo 写 config.local.yml 可以；真实平台不能指望用户去改 properties。

用户不配文件，一般是：

（1）先在平台里登记主机（Web 页面 / 管理后台）
存：主机名、IP、账号、密码（或密钥）
落库时密码要加密 / 进密钥库（Vault、KMS），不是明文 YAML。


（2）用户提问时只说业务对象
	1）「查一下 linux-231 磁盘」
	2）或界面上先选中一台机器再问


（3）模型只拿到「主机标识」（如 linux-231 / hostId=3）
	 拿不到密码。

（4）Tool 在服务端查库
hostId → 取出 IP/账号/密码 → SSH 执行 → 把结果回给模型。


//=========================
//=========================

用户：「查 231 磁盘」
    ↓
模型决定调用 Tool(host_id=linux-231)   ← 只有 ID，没有密码
    ↓
Tool 服务端：查资产表 / 密钥库 → SSH
    ↓
把 df 结果返回给模型 → 模型总结给用户
