//================================================================== 它是一个（名字 / 总概念）

ReAct = Reasoning（推理）+ Acting（行动）
//取两个词的前几个字母拼在一起，不是普通英文单词。

//================================================================== ReAct 三要素
//================================================================== ReAct 三要素

ReAct = Thought（思考）→ Action（行动）→ Observation（观察）


每一轮循环固定三步：

	Thought：	我现在该做什么？为什么？ 
	 Action：   调用哪个工具？参数是什么？
Observation：	工具返回了什么结果？

【Observation 的结果会进下一轮 Thought】

然后进入下一轮 Thought，直到可以给出 Final Answer。

//================================================================== ReAct 和 【推理】 的关系
//================================================================== ReAct 和 【推理】 的关系

（1）Thought = 推理本身（显式「想」）

（2）Action + Observation = 推理后的行动和反馈

（3）多轮循环 = 带着前面结果，一步步推下去的完整过程


//================================================================== ReAct 和 【规划】 的关系
//================================================================== ReAct 和 【规划】 的关系

//========= 规划（Planning）： 先拆大步骤（路线图）
//========= 规划（Planning）： 先拆大步骤（路线图）


（1）管什么	： 大任务拆几步
（2）粒度		： 粗：路线图
（3）例子		： ①连机 → ②查盘 → ③查目录 → ④结论
（4）类比		： 导航设途经点

//========= ReAct 循环： 逐步执行，每步可能调整
//========= ReAct 循环： 逐步执行，每步可能调整


（1）管什么	： 每一步怎么干、干完怎么办
（2）粒度		： 细：当前这一步
（3）例子		： 现在用 df 还是 du？结果不对换命令吗？
（4）类比		： 开车时看路况随时变道
 
//========= 总结
//========= 总结
规划 = 「要干什么、大致顺序是什么」
ReAct = 「正在干这一步，想→做→看，不对就改」

 
//================================================================== 总结
//================================================================== 总结

规划、ReAct、推荐 三者其实是：
先有规划、然后才有ReAct，ReAct有3要求，
其中一个要求是推理，另外两个要素是 推理后的行动和反馈
