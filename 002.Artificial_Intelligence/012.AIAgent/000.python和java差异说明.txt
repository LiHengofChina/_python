（12）java中有静态，就是在编译的时候就决定了的东西（类、方法、变量等）,python这边有吗

	有类似的，但不像 Java 那么“编译期定死”。
	（1）Java		static 字段/方法 编译期就绑定好
	（2）Python 近似	类属性、@staticmethod 仍偏动态，多在运行时决定

	Python 没有 Java 那种严格的“编译期静态绑定”；
	需要“属于类、不依赖实例”时，用类属性或 @staticmethod 即可。
	类属性大家共用一份，概念上就相当于 Java 的 静态成员变量。
	方法要用 @staticmethod（或 @classmethod）才更像静态方法。

（10）invoke 在java里面很多设计模式（好像有代理模式）都使用了 这个名字，它是意思，在哪些方常用?

	invoke =「调用 / 触发执行」，不是某个模式专有名词。

	【反射】 Java Method.invoke(obj, args)：动态调用方法
	【代理】 代理对象最终还是 invoke 去调真实方法（JDK 动态代理里就有 InvocationHandler.invoke）
	【RPC / 远程】 客户端 invoke 一次远程服务
	【框架 API】 LangChain llm.invoke(...)：把输入交给模型跑一遍
	【一般口语】 “invoke a function” = 调用函数

	银行里已经有不少用 Python 的：风控模型、数据分析、AI/Agent 平台、内部工具、部分 API 服务。

（8）python也是像java一样由jvm虚拟机管理内存吗？

	Java：JVM、JVM 管堆、GC
	CPython 解释器：引用计数 + GC

	解释器不只是“把代码翻译执行”，
	还负责对象创建、引用跟踪、垃圾回收等运行时工作。
	名字偏“解释”，实际是一"整套运行环境"。


（7）我访问 http://127.0.0.1:8101/ 地址时
	也是发送一个http，无状态的，后端处理了就结束了

	浏览器访问 http://127.0.0.1:8101/ 就是一次 HTTP 请求：要页面 → 后端返回 index.html → 这次请求结束。
	HTTP 本身是无状态的：这次请求不自动带着“上次聊了什么”。
	你后面点发送，又是一次新的 POST /api/chat；现在 Demo 没做会话记忆，所以每次都是独立问答。


（6）为什么每个文件夹下面都有一个 __init__.py 呢

	为了告诉 Python：这个文件夹是一个包（package），
	里面的模块可以用 from app.xxx import ... 导入。

	没有 __init__.py（老规矩下）它只是普通目录，导入可能失败或不稳定。
	文件可以是空的，也可以写包初始化代码；你现在空着就行。

	一句话：__init__.py = 「这是 Python 包」的标记。

（4）ddd领域在python这边也是标准的、流行的、业界认可？
	算业界认可的一种做法，但不是 Python 唯一“标准”。
	（1）分层（domain / application / infrastructure / interfaces）在 中大型、要对标 Java/DDD 的项目里很常见，完全正规。

	（2）Python / FastAPI 社区很多小项目更扁平：routers/ + services/ + main.py，也很流行。

	（3）Python 也做大型项目（Instagram、Dropbox、很多 AI/数据平台后端都是）。

	（4）分层少见，多半是因为很多 Python 项目起步小、先扁平；项目变大后一样会拆层。


（3）像spring那样的注入功能，在python这边有吗，简
	有，但不如 Spring 那么「全家桶」。

	FastAPI：用 Depends() 做依赖注入（够用）

	它也是为了提前把对象注入进来，不用自己new是吧

（2）pyhon这边有java那边的注解那种用法吗
	有，最接近的是 装饰器（decorator），写法是 @xxx。
	@app.get("/api/chat")      # ≈ @GetMapping
	def chat_get(q: str): ...
	可以这么理解：装饰器 ≈ Java 注解，都是给代码加标记，让框架自动干活。


（1）python这边不需要先写接口，再写实现这种写法吧？
	Java 里常先写接口再写实现，是因为静态类型 + 依赖注入习惯。
	Python 更常见：先直接用具体类（比如 OllamaChatGateway），
	真有第二个实现（Mock、换厂商）再抽接口。
