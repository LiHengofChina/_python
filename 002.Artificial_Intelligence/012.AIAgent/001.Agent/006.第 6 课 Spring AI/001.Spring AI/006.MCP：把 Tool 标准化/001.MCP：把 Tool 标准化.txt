MCP = Tool 的「USB 标准」：同一套 Tool，
Cursor / Spring AI / 其他客户端都能连。

	/**
		意思是：我写的工具也能让Cursor使用，让别人使用
		所以 "开源那些人" 写的工具 "我也能用"

		对。MCP 就是双向互通。

	*/

//===================================================================
//===================================================================

有点像 JAR。 

JAR 全称 Java Archive（Java 归档包），用来打包分发代码；

MCP 全称 Model Context Protocol（模型上下文协议），用来标准化 Tool 的暴露与调用。

//===================================================================
//===================================================================
MCP Server（微服务）     ←  实现 Tool：SSH、查监控、发邮件…
       ↑ MCP 协议
Agent / Cursor / 其他客户端  ←  只负责对话 + 调 Tool，不写具体业务

//===================================================================
//===================================================================

Tool 代码 集中在 MCP Server（可按领域拆多个微服务：运维 MCP、数据库 MCP…）
Agent 客户端 通过 MCP 发现并调用，不必把 Tool 写进 Agent 进程

多数是 独立进程/服务，不是 Java 依赖 jar。
