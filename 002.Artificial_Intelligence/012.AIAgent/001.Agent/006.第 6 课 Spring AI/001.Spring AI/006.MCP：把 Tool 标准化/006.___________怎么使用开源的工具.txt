spring:
  ai:
    mcp:
      client:
		//============= （第一种配置方法）连已经跑起来的 MCP 服务（你要先启动 8104，Client 再去连）
        sse:
          connections:
            ops-server:
              url: http://127.0.0.1:8104       # 你的
            weather-server:
              url: http://localhost:8080       # 开源/别人的 HTTP MCP #远程已启动的服务（HTTP），如你的 8104

		//============= （第二种配置方法）自己启动时带一个MCP起来， Client 启动时用 npx 拉起一个 Node 进程跑 filesystem MCP。 需 Node.js + npx
        stdio:
          connections:
            filesystem:
              command: npx
              args:
                - -y
                - "@modelcontextprotocol/server-filesystem"
                - "E:\\_python"

//============================================================================ 本地怎么搞
//============================================================================ 本地怎么搞

STDIO 常见场景： Cursor、Claude Desktop 等本机工具，Client 拉起本地进程，不依赖远程 HTTP。

内网/不能联网：

SSE + 内网 MCP 服务（像你的 8104）同样适用——走内网 HTTP，不需要公网
STDIO + npx 第一次还要联网下载包；之后缓存了才能离线用

//============================================================================ 生产怎么搞

外网：mvn package → day06-mcp-server.jar
内网：java -jar day06-mcp-server.jar
Agent：sse.connections.url → http://内网IP:8104
