
//================================================================== ReAct和「规划」的关系
//================================================================== ReAct和「规划」的关系
	┌──────────┐
	│   用户    │  输入目标 + 连接信息
	└────┬─────┘
		 ↓
	┌──────────┐
	│  Agent   │  协调中心（循环控制）
	│  编排层   │
	└────┬─────┘
		 ↓
	┌──────────┐     ┌─────────────┐
	│ Planner  │────→│   Memory    │  短期：本轮步骤/结果
	│  规划器   │     │   记忆       │  长期：历史案例/偏好
	└────┬─────┘     └─────────────┘
		 ↓
	┌──────────┐
	│   LLM    │  大脑：Thought、选 Tool、生成 Final Answer
	└────┬─────┘
		 ↓
	┌──────────┐
	│  Tools   │  手和脚
	│          │  ├─ SSH 执行命令
	│          │  ├─ RAG 查手册
	│          │  ├─ 监控 API
	│          │  └─ 告警通知
	└────┬─────┘
		 ↓ Observation 回传给 LLM
		 ↓（循环直到完成）
	┌──────────┐
	│  结果输出 │  报告 + 建议（危险操作等人审批）
	└──────────┘

//============================== 各组件一句话
//============================== 各组件一句话

	用户				//提目标、配连接、审批
	Agent 编排层		//控制 ReAct 循环、调 LLM、调 Tool
	Planner			//拆步骤（可和 LLM 合一）
	LLM				//推理、决策、生成
	Memory			//记住已做步骤和上下文
	Tools			//执行真实操作
	RAG				//Tools 里的一种（查文档）

//==================================================================
//==================================================================

用户配置连接对象（主机、凭证）
        ↓
Agent 编排层接收任务
        ↓
LLM 规划 + ReAct 循环
        ↓
Tool：SSH 查日志/指标 + RAG 查手册
        ↓
输出：根因 + 方案
        ↓
生产操作 → 人工点击执行（Tool 加权限/审批）
