
//================================================================== 规划（Planning）
//================================================================== 规划（Planning）

Agent 把大目标拆成小步骤。

目标：定位 XXX 机器磁盘告警
  → 步骤1：连接主机
  → 步骤2：查磁盘使用率
  → 步骤3：找大目录
  → 步骤4：查是否有异常进程
  → 步骤5：汇总原因 + 给建议

/**
单次问答：不会自动拆步骤，你问什么它答什么
Agent：自己规划「先做什么、后做什么」
*/

//================================================================== 工具（Tool / Function Calling）
//================================================================== 工具（Tool / Function Calling）

Agent 的「手和脚」——能调用外部能力。

//=========== 工具例子
//=========== 工具例子

SSH 执行命令		//df -h、du -sh
查监控 API		//拿 CPU、内存曲线
查日志库			//搜 ERROR 关键字
RAG 检索			//查运维手册
发告警			//通知值班人

/**
单次问答：只能基于 Prompt 里已有的文字「说」
Agent：可以 真的去查、去执行（在你允许范围内）
*/

//================================================================== 记忆（Memory）
//================================================================== 记忆（Memory）
Agent 记住上下文，多步任务才能连贯。

（1）短期记忆：本轮对话：刚查到磁盘 95%，下一步查 /var/log
（2）长期记忆：上次处理过同类问题、用户偏好、历史案例

//===========
//===========

（1）单次问答：通常只有当前 Prompt（+ 你手动带的历史）
（2）Agent：显式维护「已经做了什么、得到了什么」

//================================================================== 循环（Loop / ReAct）
//================================================================== 循环（Loop / ReAct）

Agent 反复：思考 → 行动 → 观察结果 → 再思考，直到完成或达到上限。

循环 1：思考「先连机器」 → 行动 SSH → 观察「连接成功」
循环 2：思考「查磁盘」   → 行动 df -h → 观察「/ 分区 95%」
循环 3：思考「查大目录」 → 行动 du    → 观察「/var/log 占 40G」
循环 4：思考「够了，给结论」→ 输出答案 → 结束

/**

循环指 Agent 整体的 「想 → 做 → 看结果 → 再想」，一轮一轮重复，直到任务完成。

*/

/**
ReAct = Reasoning（推理）+ Acting（行动）

Agent 交替做两件事：

想：下一步该干什么。
做：调用工具执行，看结果。

循环直到任务完成。就是上面「思考 → 行动 → 观察」那种模式。

*/

//================================================================== 四个核心
//================================================================== 四个核心
四个核心：规划、工具、记忆、循环

ReAct 是循环的一种 "具体模式（推理+行动）"
