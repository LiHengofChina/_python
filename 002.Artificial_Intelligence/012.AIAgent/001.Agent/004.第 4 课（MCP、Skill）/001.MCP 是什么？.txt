//======================================================================= MCP 是什么？

MCP = Model Context Protocol（模型上下文协议）

一句话：用 "统一标准" ，把 Tool、数据、资源暴露给大模型。

//======================================================================= 为什么需要 MCP？
//======================================================================= 为什么需要 MCP？

//============ 没有 MCP 时：
//============ 没有 MCP 时：

	Spring AI 里注册一套 Tool
	LangChain4j 里又注册一套
	Cursor 里再写一套
	→ 各搞各的，不能复用

//============ 有 MCP 时：
//============ 有 MCP 时：

运维平台暴露 MCP Server（SSH、RAG、监控…）
    ↓
Cursor / Spring AI / 其他 Agent 客户端
    ↓
同一套协议连接，同一批 Tool 都能用

MCP = Tool 的「USB 标准」——接口统一，换客户端不用重写 Tool。

//======================================================================= MCP 和 Tool 的关系
//======================================================================= MCP 和 Tool 的关系

（1）Tool：	单个能力（函数）			//具体能力
（2） MCP：  暴露 Tool 的协议/规范		//把多个 Tool 打包成 Server 供外部连接

//=========== 流程
//=========== 流程
你的 Java 代码
    → 实现 ssh_exec、rag_search（Tool）
    → 用 MCP Server 暴露出去
    → Agent 客户端通过 MCP 调用

// Cursor 里用的 MCP，就是这个思路。
