//================================================== 和 Tool、MCP 的分工

Skill = 一块「专长包」（知识、步骤、模板），告诉 Agent 某类事怎么做。

规划、ReAct 循环 = Agent 底层运行机制，Skill 是在这个机制上 被选用、被注入 的内容，不是替代它。

//==================================================
//==================================================

//大多数 Skill 
		正文 → 进 system prompt
		description → 多 Skill 时用来选哪个 Skill（路由）
		tools → 决定绑定哪些 Tool（不全是提示词）
		不全是「只改提示词」
		还有：

//还有：
		限制能用哪些 Tool（tools_for_skill）
		多 Skill 时先路由再加载
		有的平台还会挂脚本、资源文件

//==================================================
//==================================================

一句话：Skill 核心是 塞进 system 侧指令，用户内容单独发；
同时往往还管 Tool 清单 和 选中哪个 Skill。
