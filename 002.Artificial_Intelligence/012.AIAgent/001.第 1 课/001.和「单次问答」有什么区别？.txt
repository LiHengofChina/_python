//================================================================== Agent 是什么？

Agent = 大模型 + 目标 + 工具 + 记忆 + 循环执行

不只 "「回答问题」" ，而是 为了 "完成目标"，自己决定 "下一步" 做什么。

//============
//============
用户：帮我查 XXX 机器磁盘为什么满，并给出处理建议
    ↓
Agent 思考：需要先连服务器 → 执行 df -h → 再看 /var/log
    ↓
调用 Tool：SSH 执行命令
    ↓
拿到结果，再思考：日志太大 → 建议清理
    ↓
（可能多轮 Tool 调用）
    ↓
输出：根因 + 建议（生产操作等人审批）

/***
核心区别：不是答一次就停，而是「想 → 做 → 看结果 → 再想 → 再做」直到任务完成。
*/


//================================================================== 单次问答
//================================================================== 单次问答

用户：磁盘满了怎么办？
    ↓
（可选 RAG 检索资料）
    ↓
LLM 生成一段回答
    ↓
结束

/***

特点：
	一步完成
	不会自己再查、再试、再执行
	没有「记忆上次聊了什么」（除非你把历史手动塞进 Prompt）

	这就是 ChatGPT 普通对话、
*/
//================================================================== 两者对比
//================================================================== 两者对比

（1）调用次数
		 单次问答： 通常 1 次
		  Agent： 多轮循环

（2）会不会规划
		 单次问答： ❌ 不会自动规划
		  Agent： ✅ 拆步骤

（3）会不会用工具
		 单次问答： ❌（除非框架帮你封装）
		  Agent： ✅ SSH、API、RAG 等

（4）有没有记忆
		 单次问答： 弱（靠 Prompt 历史）
		  Agent： ✅ 短期/长期

（5）输出
		 单次问答： 一段文字
		  Agent： 完成任务 + 文字报告

（6）例
		 单次问答： RAG 问答 Demo
		  Agent： 运维自动排查
