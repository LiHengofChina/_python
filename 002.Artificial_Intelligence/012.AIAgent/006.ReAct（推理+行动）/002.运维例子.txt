用户目标： 查 XXX 机器磁盘为什么满

【循环 1】
	Thought:      需要先连上目标主机才能查磁盘
	Action:       ssh_exec(host="XXX", command="...")
	Observation:  连接成功

【循环 2】
	Thought:      连接成功，先查各分区使用率
	Action:       ssh_exec(host="XXX", command="df -h")
	Observation:  / 分区 95%，/var 80%

【循环 3】
	Thought:      / 分区很高，查哪个目录占用大
	Action:       ssh_exec(host="XXX", command="du -sh /var/log/*")
	Observation:  /var/log/app 占 40G

【循环 4】
	Thought:      根因是日志过大，查手册看标准处理方式
	Action:       rag_search(query="磁盘满 日志清理")
	Observation:  手册：先 logrotate，禁止 rm 正在写入的文件

【循环 5】
	Thought:      信息够了，可以给出结论
	Final Answer: 根因：/var/log/app 日志未轮转占 40G；
				  建议：执行 logrotate…（生产环境需人工审批）

LLM 输出通常是结构化 JSON 或固定格式：

Thought: 我需要查磁盘
Action: ssh_exec
Action Input: {"host":"XXX","command":"df -h"}
---
Observation: Filesystem  Size  Used ... 95%

框架（Spring AI / LangChain4j）负责：
	1） 解析 Action → 
	2） 执行 Tool → 
	3） 把 Observation 塞回 Prompt → 再调 LLM。
