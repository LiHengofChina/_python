## 四、标准化优化建议

### 🔴 紧急处置建议（P0）

#### 1. 开启二进制日志（两个实例均需执行）

**影响**：无法实现数据恢复、不满足等保合规要求

**执行步骤**：

```bash
# 1. 编辑 MySQL 配置文件 my.cnf
[mysqld]
server-id = 1                    # test1 设置为 1，test2 设置为 2
log_bin = mysql-bin
binlog_format = ROW
expire_logs_days = 7
max_binlog_size = 100M
sync_binlog = 1

# 2. 重启 MySQL 服务
systemctl restart mysqld

# 3. 验证配置
mysql -e "SHOW VARIABLES LIKE 'log_bin';"
mysql -e "SHOW MASTER STATUS;"
```

**预期效果**：实现数据恢复能力，满足等保合规要求

---

### 🟠 重要优化建议（P1）

#### 2. 收紧全网通配符账号权限（两个实例均需执行）

**影响**：不符合等保合规要求，存在安全风险

**执行步骤**：

```sql
-- 1. 查看当前账号
SELECT user, host FROM mysql.user WHERE host='%';

-- 2. 创建受限账号（推荐）
CREATE USER 'root_admin'@'192.168.50.%' IDENTIFIED BY 'StrongPassword123!';
GRANT ALL PRIVILEGES ON *.* TO 'root_admin'@'192.168.50.%' WITH GRANT OPTION;

-- 3. 删除全网通配符账号（谨慎操作）
DROP USER 'root'@'%';

-- 4. 针对监控账号，限制为特定 IP
CREATE USER 'exporter'@'192.168.50.100' IDENTIFIED BY 'exporter_password';
GRANT PROCESS, REPLICATION CLIENT, SELECT ON *.* TO 'exporter'@'192.168.50.100';
DROP USER 'exporter'@'%';

-- 5. 刷新权限
FLUSH PRIVILEGES;
```

**预期效果**：符合等保合规要求，降低安全风险

### 🟡 性能优化建议（P2）

#### 3. 分析并优化慢查询（test1 实例）

**影响**：数据库性能劣化，影响业务响应时间

**执行步骤**：

```bash
# 1. 查看慢查询日志配置
mysql -e "SHOW VARIABLES LIKE 'slow_query%';"

# 2. 使用 pt-query-digest 分析慢查询
pt-query-digest /var/lib/mysql/mysql-slow.log > slow_analysis.txt

# 3. 查看执行计划
mysql -e "EXPLAIN SELECT * FROM your_table WHERE ...;"

# 4. 添加缺失索引
mysql -e "ALTER TABLE table_name ADD INDEX idx_column (column_name);"
```

#### 4. 优化全表扫描问题（test1 实例）

**执行步骤**：

```sql
-- 1. 查看哪些 SQL 在全表扫描

-- 2. 检查缺失索引的表

-- 3. 根据查询条件添加索引
```
