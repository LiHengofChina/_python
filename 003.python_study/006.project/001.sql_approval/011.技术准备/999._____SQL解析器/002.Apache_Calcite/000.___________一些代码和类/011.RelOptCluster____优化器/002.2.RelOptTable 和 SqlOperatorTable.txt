
RelOptTable 和 SqlOperatorTable 在概念上有一定相似性，
但它们的作用和应用场景不同。
简单来说，它们都像是 "表"，但是处理的内容不同。
//===================== SqlOperatorTable
//===================== SqlOperatorTable

作用：SqlOperatorTable 用来管理和定义 SQL 操作符和函数。它
	  是 SQL 语言中的操作符（如 +, UPPER）和函数的容器，
	  提供查询、管理和调用 SQL 中所有操作符的功能。

用途：它通常用于处理 SQL 解析过程中 "操作符" 和 "函数" 的 "匹配和使用"。

//===================== RelOptTable
//===================== RelOptTable

作用：RelOptTable 是 查询优化器中的表，
	  它表示一个 "数据库表" 或 "视图"。
	  在查询优化阶段，RelOptTable 提供有关 "表或视图" 的 "元数据信息"，
	  比如表的列、类型、是否为 "聚合表" 、是否 "分区" 等。

用途：它在 "查询的优化过程" 中作为 "查询计划" 的一部分，
	  帮助 "优化器" 理解和操作 "数据库表"。
	  RelOptTable 的信息帮助优化器 "生成和优化" 查询计划。


//========================================= 相似性：
//========================================= 相似性：
（1）"表"：都可以被认为是 某种形式的 "表"，
			但它们分别 "处理 SQL 操作符" 和 "数据库表/视图" 的定义。

（2）容器：它们都是容器，分别容纳了 "SQL 操作符/函数（SqlOperatorTable）" 或 "数据库表（RelOptTable）的信息"。


//========================================= 区别：
//========================================= 区别：
（1）SqlOperatorTable：专注于管理和提供 SQL 语法中的操作符和函数。
（2）RelOptTable：专注于数据库表（或视图）的元数据，供查询优化器使用。


总结来说，SqlOperatorTable 和 RelOptTable 都是“表”的概念，
但它们处理的内容和用途完全不同，分别用于 SQL 操作符和查询优化中的数据库表管理。
