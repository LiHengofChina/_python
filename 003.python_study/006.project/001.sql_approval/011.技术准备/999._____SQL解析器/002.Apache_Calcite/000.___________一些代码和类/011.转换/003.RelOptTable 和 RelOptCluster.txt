
RelOptTable 和 RelOptCluster 没有直接的继承或组合关系，
但它们在 查询优化 过程中是协同工作的：

//========================================
//========================================

（1）RelOptTable 代表 数据库表，存储表的 Schema、字段类型、统计信息，供优化器使用。

（2）RelOptCluster 代表 查询优化的上下文，
	 管理 优化器 (RelOptPlanner)、RexBuilder、RelMetadataProvider，
	 用于创建和优化 RelNode（查询计划）。


//======================================== 它们的关系
//======================================== 它们的关系
（1）在 SqlToRelConverter 过程中，
	 RelOptCluster 会访问 RelOptTable 获取表的元数据信息，
	 以便优化 SQL 查询。

（2）当优化器 (RelOptPlanner) 选择最佳执行计划时，
	  它会通过 RelOptCluster 调用 RelOptTable 获取表的行数、索引、分区等统计信息，
	  从而决定最优的执行路径。

//======================================== 简单来说：
//======================================== 简单来说：

✅ RelOptTable 提供 表的元数据，RelOptCluster 组织查询优化过程，
它们 在 SQL 优化阶段协同工作。 🚀
