对于你的 SQL 审核平台，
如果审核目标数据库的操作不频繁，
理论上你可以选择 不使用连接池，但这取决于以下几个因素：

//==================================== 是否需要连接池管理？
连接池的作用：
	（1）连接池用于管理数据库连接的创建、复用和销毁，目的是提升性能，
		 尤其是对于 频繁的数据库操作。
		 它可以避免每次访问数据库时都创建新连接，
		 从而节省连接建立和销毁的时间开销。


不频繁的操作：
	（1）如果你只进行偶尔的查询或执行一些较少的审核操作，
		 且每次操作的数据库连接数量不大，可以在每次需要时动态获取连接，
		 而不使用连接池管理。这种方式简单、直接，
		 并且不必担心池中的连接过多占用资源。


使用连接池的场景：
	（1）如果审核平台 有较高并发 或需要连接多个目标数据库，
		使用连接池将有助于提高性能和资源管理。
		但如果只是偶尔执行一次 SQL 查询（比如检查某些规则、获取执行计划），
		使用连接池的性能优势可能不太明显。

总结：
	（1）如果你的 SQL 审核平台 只是偶尔与数据库交互，且没有高并发要求，你可以不使用连接池，每次手动获取数据库连接即可。

	（2）但如果有 多次频繁的数据库交互 或 多个并发用户，则建议使用 连接池（例如 HikariCP 或 C3P0）来管理数据库连接，提升性能和稳定性。

根据你的需求，选择是否使用连接池。
如果不确定，可以先试用不使用连接池的方式，待需求增长时再引入连接池。
