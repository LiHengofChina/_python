
//============================================ Calcite
//============================================ Calcite
		Calcite 的 SQL 解析器主要基于 ANSI SQL-92（SQL-92）标准实现，
		但并不限于 SQL-92。
		它的核心语法框架来自 SQL-92，
		但在此基础上又扩展和融合了多种 SQL 方言。

//============================================ Calcite 的 SQL 语法来源是什么？
//============================================ Calcite 的 SQL 语法来源是什么？

Calcite 的 parser（SqlParser）并不是完全自己造的，而是：

✔ 基于早期的 Farrago SQL parser
✔ 而 Farrago SQL 是严格遵循 ANSI SQL-92（SQL2）标准语法
✔ Calcite 在此基础上继续发展

	因此：
		🎯 Calcite 的核心语法是 ANSI SQL-92，但并非只支持 SQL-92

//============================ 🔍 Calcite 覆盖的 SQL 标准包括：
//============================ 🔍 Calcite 覆盖的 SQL 标准包括：

1） 主体框架：SQL-92 （最重要）
	例如：
		INSERT 必须写成 INSERT INTO
		JOIN 语法遵循 SQL92
		SELECT/FROM/WHERE/ORDER/DISTINCT 等经典语法
		DDL 不完整（因为 Calcite 主要做解析）

2） 扩展支持：SQL:1999 （部分）

	如：
		WITH 语句
		窗口函数（OVER, PARTITION BY）

3） 再向前的部分扩展：SQL:2003 / SQL:2011 （少量）
	如：
		MERGE（部分）
		部分 JSON 支持
		但 Calcite 不完全支持 SQL2003+ 的所有规范。

//============================ 💡 Calcite 还内置了很多方言支持（这超出了 SQL92）
//============================ 💡 Calcite 还内置了很多方言支持（这超出了 SQL92）

例如：
	MySQL 方言（部分支持）
	Hive 方言
	BigQuery 方言
	Spark SQL 方言
	Oracle 方言
	PostgreSQL 方言

这些都来自 Calcite 的不同 SqlDialect。

//============================================ 总结
//============================================ 总结

Calcite 的核心语法是基于 ANSI SQL-92 标准实现的，
但它同时支持多个 SQL 方言，
是一个结合标准 SQL + 多方言扩展的混合解析器。
