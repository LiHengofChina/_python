
（1）生成 PDF
	 用户更习惯“报告”以 PDF 形式阅读并归档。
	 可在渲染好 HTML 后，用工具（如 wkhtmltopdf、Puppeteer/Playwright Headless Chrome）将 HTML 转成 PDF，再返回给前端下载。


（2）图表和数据表
	 如果报表里有图表，建议用 ECharts 并把图表的初始化脚本写在模板里。
	 数据量大时，可分页渲染或仅导出汇总表格。


（3）样式管理
	 虽然“单文件”方便携带，但后期维护不便。可考虑把通用样式提取到公共 CSS，模板里只引入外部 CDN 地址。


（4）安全与性能
	 防止模板注入，Freemarker 默认会对 ${…} 进行 HTML 转义。
	 如果数据量非常大，渲染前端一次性插入可能卡顿，可后端分页或分块加载。


（5）版本和多份报告
	 如果需要保留历史报告，可把渲染好的 HTML/PDF 存到对象存储，再生成带版本号的下载链接。
