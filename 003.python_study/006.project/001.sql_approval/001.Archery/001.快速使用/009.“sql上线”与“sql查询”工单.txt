

在 Archery 系统中，"SQL上线工单" 和 "SQL查询工单" 是两个核心功能模块。


它们主要围绕  "数据库操作的安全性"、"合规性" 和 "管理流程" 展开。


它们主要用来管理和审核线上系统中涉及的 SQL 操作（增、删、改、查）的工具，
主要场景是 "新功能上线前或运行中执行 的 SQL " 需要审核，确保安全性、稳定性和合规性，
避免对生产环境造成影响。



SQL上线工单：用于管理和审批涉及数据库变更的操作（如增、删、改），防止对生产环境造成破坏性影响。
SQL查询工单：用于管理和审批数据库的查询操作（如 SELECT），防止数据泄露或高负载查询影响性能。

//============================================================================================================================ 工单 "上线和查询" 的审批流程


（1）SQL上线工单：例如创建表、修改表结构、插入、更新或删除数据等。
	 流程：
		（1）用户提交SQL上线申请工单。
		（2）系统对SQL进行语法检查、风险评估（如检测是否有大表更新、全表扫描等）。
		（3）提交审批流，涉及的审批人可能包括开发负责人、DBA（数据库管理员）、甚至业务负责人。
		（4）审批通过后，系统执行SQL语句，通常会有执行计划评估和变更后的结果校验。

（2）SQL查询工单：通常是SELECT语句），以避免直接访问数据库带来的风险。
	 流程：
		（1）用户提交SQL查询申请工单。
		（2）系统对查询语句进行检查（如是否包含敏感字段、是否带有全表扫描等）。
		（3）提交审批流，涉及的审批人可能是DBA或业务负责人。
		（4）审批通过后，系统以受控的方式执行查询并返回结果。
