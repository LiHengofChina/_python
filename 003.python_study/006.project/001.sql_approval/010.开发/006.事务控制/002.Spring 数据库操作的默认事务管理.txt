Spring 数据库操作的默认事务管理

（1）Spring 会为每个由 @Repository 或 
	  @Service 注解标记的类中的数据库操作
	  （即对 JpaRepository、CrudRepository 等接口的调用）自动开启事务。


例如，你在 auditParameterRepository.updateAuditParameter 方法中执行更新操作时，
	  Spring 会默认为这个方法加上一个事务
	  （前提是该方法调用发生在服务类中，且该服务类已经被 Spring 管理）。


//============================================（）什么时候不需要显式使用 @Transactional：
//============================================（）什么时候不需要显式使用 @Transactional：

（1）
如果你的方法只是执行一个简单的数据库操作（例如增、删、改），
并且该方法所在的类被 Spring 管理（如标记为 @Service 或 @Repository），
Spring 会自动为其创建事务。


（2）
只有在处理复杂事务逻辑、跨多个方法或多个数据库操作时，
显式使用 @Transactional 注解才显得必要，
特别是当你需要控制事务的传播行为、隔离级别、回滚条件等时。
