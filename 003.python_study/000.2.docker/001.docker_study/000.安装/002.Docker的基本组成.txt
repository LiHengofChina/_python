Docker 主要分为 客户端 和 服务器端



但它不是 "传统的 CS 架构"。
	它是一个 "分布式的容器" 化平台，
	"客户端和服务器端" 是通过 "Docker API 通信" 的。


//========== Docker Daemon（服务端）
//========== Docker Daemon（服务端）
 
	 是实际运行容器的部分
	 包含 Docker 引擎（Docker Daemon），
	 负责容器的创建、运行、管理等。
	 安装后，它会在后台 "持续运行"。

//========== Docker CLI（客户端）：
//========== Docker CLI（客户端）：
 
	 是你用来与 "Docker 服务器端" 交互的 "命令行工具（docker 命令）"。
	 你通过它来发起指令，如创建、启动、停止容器等。
