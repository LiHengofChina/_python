
尽管它们的 "名称相似"，都是作为系统或应用的基础模板。


//====================== 概念上的区别
//====================== 概念上的区别

Docker 镜像：是一个只读的模板，包含了运行应用程序所需的操作系统环境、依赖库、配置文件等。
		    Docker 镜像通常非常轻量，
		    包含了构建和运行某个应用的必需文件，
		    而不包含完整的操作系统。
		    它基于 "容器技术"，
		    容器运行时与宿主机共享操作系统内核，因此比虚拟机 "更轻量、更高效"。


虚拟机镜像：是一个完整的操作系统（或操作系统的快照），它包含了操作系统的内核、系统库、应用程序等内容。
			虚拟机镜像通常会占用大量的磁盘空间，因为每个虚拟机都需要一个完整的操作系统，
			它们运行在一个独立的虚拟化环境中，虚拟机之间是相互隔离的。

//====================== 运行机制的不同
//====================== 运行机制的不同

Docker 镜像：Docker 是基于容器技术的，
		     容器运行时共享宿主机的操作系统内核。
			 每个容器都非常轻量，启动速度非常快，因为它不需要启动一个完整的操作系统，
			 只需要加载 Docker 镜像中的应用和库等环境。
			 而且，容器是与宿主机系统共享内核的，
			 容器之间的隔离是通过操作系统级别的虚拟化（如 Linux namespaces 和 cgroups）来实现的。

虚拟机镜像：虚拟机镜像通常包括一个完整的操作系统（包括内核），
		     它运行在虚拟化软件（如 VMware、VirtualBox、Hyper-V）上。
		     每个虚拟机都需要加载一个完整的操作系统，并且每个虚拟机有自己独立的内核。
		     虚拟机通常是较为重量级的，启动时间较长，资源开销较大，
		     因为每个虚拟机都有完整的操作系统。



//====================== 资源占用
//====================== 资源占用
Docker 镜像：
			由于容器共享宿主机操作系统的内核，
			因此它们非常轻量，并且启动非常迅速。多
			个容器可以共享同一个宿主机的内核，节省资源开销。
虚拟机镜像：每个虚拟机都需要一个完整的操作系统和内核，
			这使得虚拟机通常占用更多的资源（CPU、内存、磁盘等）。
			虚拟机需要虚拟化软件的支持（例如 VMware、VirtualBox），
			并且每个虚拟机启动时需要加载完整的操作系统。

//====================== 启动速度
//====================== 启动速度

Docker 镜像：
		启动非常快，因为容器只需要加载应用程序和相关依赖，
		不需要加载完整的操作系统。

虚拟机镜像：
		启动较慢，因为每个虚拟机都需要加载整个操作系统，
		并且启动时还需要启动虚拟化层。
